# 骆驼祥子 · 命运测试 — 设计文档

**日期**：2026-04-18
**作者**：Javen + Misty
**状态**：设计确认中

## 背景

灵感来自 `https://www.readinglife.com/meks.html`（百年孤独版命运测试）——一个两幕式文学主题人格测试页面。本项目复刻其交互模式，但换成老舍《骆驼祥子》，并将基调从"诗意命运"改为"黑色幽默·当代共鸣"，让用户在戏谑中照见现代生活状态。

## 目标

- 一个**纯静态**的单页 Web 应用，用户在手机上 3-5 分钟完成测试，得到一个结果卡 + 命运短文。
- 部署至 Cloudflare Pages，主要传播渠道为朋友圈 / 小红书。
- 无后端、无追踪、无登录，所有逻辑在客户端完成。
- 内容与逻辑解耦：题目/原型/文案集中在 `content.js`，方便后续增改。

## 非目标

- 不做用户账号 / 保存历史记录。
- 不做后端存储、排行榜、数据分析。
- 不接入微信原生 JS-SDK 分享（复杂度高、收益低）。
- 不做 i18n，只做简体中文版。

## 创作基调

**黑色幽默派**：用老北平人物和情境包装当代生存焦虑，读者在戏谑中产生共鸣。参考语感：「又是一个没拉到活儿的傍晚」隐喻"客户流失"；「攒钱买车」隐喻"攒首付/副业变现"。

```

   ┌────────────────────────┐
   │ 旁白（旧时代 flavor）："拉车三年，分文未存。"│
   │ 选项（现代化）：                          │
   │   ① 换个平台跑                           │
   │   ② 开始做副业                           │
   │   ③ 投资自己学技能                       │
   │   ④ 买彩票                               │
   └────────────────────────┘
④ 结果卡（原型名 + 一句点题 + 关键特征列表）
⑤ 命运短文（祥子视角第二人称，~200-250 字）

```

### 进度持久化

- 用 `localStorage` 存三项：当前 step、已选起点、已答题目数组
- 刷新页面 / 意外关闭后重新打开，自动回到上次进度
- 点击"重新测试"按钮清空 localStorage 并回到封面

## 计分算法

### 四维人格向量

| 维度 | 含义 | 范围 |
|---|---|---|
| **卷度**（drive） | 主动付出 vs 躺平 | 0–10 |
| **韧性**（grit） | 扛压 vs 易崩 | 0–10 |
| **算计**（calc） | 精明 vs 耿直 | 0–10 |
| **善意**（kind） | 为他人 vs 为自己 | 0–10 |

### 原型签名向量（示例）

| 原型 | drive | grit | calc | kind |
|---|---|---|---|---|
| 副业祥子 | 9 | 8 | 3 | 6 |
| 卷王虎妞 | 10 | 9 | 7 | 4 |
| 佛系老马 | 1 | 5 | 3 | 7 |
| 斜杠高妈 | 8 | 6 | 10 | 4 |
| 曹先生 | 5 | 4 | 2 | 9 |
| 刘四爷 | 7 | 8 | 10 | 1 |
| 二强子 | 1 | 1 | 2 | 2 |
| 小福子 | 6 | 3 | 3 | 8 |

（实际数值在 content.js 里定稿时微调，确保 8 个向量在四维空间中分离良好）

### 计分流程

1. 用户答题时累加每个选项的 `{drive, grit, calc, kind}` 权重
2. Q9-Q10 权重 ×1.5
3. 最终用户向量 `u = [du, gu, cu, ku]`
4. 对 8 个原型，计算欧氏距离 `d = ||u - archetype||`
5. 取最小距离的原型作为结果

### 起点对结果的影响

**不影响原型选择**（保证结果公平），**只影响命运短文的开场**。

## 分享机制

### 复制链接
- 按钮触发 `navigator.clipboard.writeText(url)`
- 复制的 URL 指向该原型的**静态分享页**：`https://xiangzi-fate.pages.dev/r/xiangzi.html`
- 这个 URL 同时承担两个职责：微信 / 钉钉预览时展示 OG 卡片；被点击后跳转到主站的结果预览

### 下载结果卡 PNG
- 使用 `html2canvas`（CDN 引入）把结果卡 DOM 渲染成 PNG
- 命名 `骆驼祥子测试-副业焦虑的祥子.png`
- 用户保存后手动发朋友圈 / 小红书

### 静态分享页（OG 预览）
- 8 个原型各一个 `r/<id>.html`
- 每个页面的 `<meta property="og:title">`、`og:description`、`og:image` 针对该原型定制
- 分享到微信群时显示缩略图+标题
- 页面 body 内容：原型卡预览 + 跳转主站按钮

### 无追踪
- 无 analytics、无 cookie、无埋点

## 部署

### 技术路径
1. 项目初始化为 git 仓库（已完成）
2. 本地开发使用 `python3 -m http.server` 或 `npx serve`
3. 完成后推送至 GitHub 新仓库 `xiangzi-fate`（public）
4. Cloudflare Pages 对接 GitHub 仓库 → 配置 `build output: /`（纯静态，无 build step）
5. 自动部署至 `xiangzi-fate.pages.dev`

### 前置条件
- 用户本地需 `wrangler login` 或 CF Dashboard 登录
- 用户需授权 CF Pages 访问 GitHub

### 自定义域名
- 本期不绑定，使用 CF 默认的 `.pages.dev` 域名
- 后续可随时在 CF Dashboard 添加

## 测试与验收标准

- [ ] 刷新页面不丢失当前进度（localStorage 存储）

- [ ] 结果卡截图效果符合"报纸剪报"气质

（本期不做深色模式）

### 性能
- [ ] 首屏 < 2s（国内移动网络）
- [ ] 无外部字体/图片加载阻塞

## 后续可能的改进

- 若 html2canvas 生成画质不理想，改为手写 canvas 绘制结果卡
- 若后续想做更精致的分享预览，可上 CF Workers 动态生成 OG 图
- 若上线后数据显示某些原型几乎抽不到，根据真实数据重新调整签名向量

## 已明确的实现约定

- 结果卡导出：**html2canvas**（CDN 引入）
- OG 图：**8 张静态 PNG**（用 html2canvas 在本地预渲染一次后提交到 `r/og/` 目录）
- 进度持久化：**localStorage**（key: `xiangzi-fate-state`）
- 分享 URL：**静态路由** `/r/<archetype-id>.html`（微信可预览 OG 卡片）
